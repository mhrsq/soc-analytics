Chart & Visualization Style Guide

Reference untuk bikin chart yang consistent across SOC Analytics dashboard. Focus: operational clarity, info density, color as signal (bukan decoration).

Dark theme only · IBM Plex Sans + Mono · Monochrome dominan + 3 signal colors

Signal colors
use sparingly · 10% usage

3 colors, 3 meanings. Red = breach / critical / failing. Amber = warning / at-risk / attention needed. Green = healthy / recovered / on-target. Never use these decoratively. Kalau ada chart tanpa problem, it's all gray.

≥ 7d window

Do & Don't 16

✕ Don't
  • Use 6+ colors for 6 KPI cards
  • Put colored icon in rounded pastel bg in every tile
  • Draw donut charts for 2-value ratios
  • Use circular gauge for a single % number
  • Render a 2-bar chart when you have 2 data points
  • Use medal / trophy / tier gamification
  • Write "Not enough data" + ghost illustration
  • Use emojis as chart indicators
  • Show percentages without the fraction
  • Leave deltas off primary metrics
  • Use monospace font for section titles
  • Use border radius ≥ 10px on panels
  • Use generic Inter font everywhere
  • Use pill-shaped progress bars
  • Show drop-shadows on dark-theme cards
✓ Do
  • Dominant gray, 3 signal colors used sparingly
  • Plain text labels, numbers in mono, no decorative icons
  • Two paired progress bars for ratios
  • Single number + sparkline + delta instead
  • Print the actual numbers as a sentence when n ≤ 3
  • Flag workload imbalance; skip competitive rankings
  • Factual empty text + when data will update
  • Colored dots (1px radius) or badges only
  • Always pair % with "X / Y"
  • Every primary metric has "[±value] · [window]"
  • IBM Plex Sans for titles, Mono only for data
  • 4–6px radius on panels, 1–3px on inline elements
  • IBM Plex Sans + Mono, or equally distinctive pair
  • Sharp 1px bar radius
  • Flat surfaces, 1px borders, no elevation shadows
